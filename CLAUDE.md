# gukin.dev — 블로그 작성 컨텍스트

Hugo + PaperMod 기반 정적 블로그. 새 글 작성/수정 시 이 문서를 먼저 읽는다.

## 글쓰기 원칙

- **흐름**: 도메인 이해 → 기술 판단 → 검증 → 회고
- **톤**: ~다체. 회고형 (시간순 + 의미 짚는 한 줄)
- **추측과 측정 구분**: "아마 X일 거다"가 아니라 "측정해보니 Y였다"
- **틀린 가설도 드러내기**: 과정의 솔직함이 글의 강점
- **마지막 단락에 회고/통찰 한 줄**: 일기를 회고로 격상시키는 공식

## 회사 데이터 사용 원칙 (중요)

회사 자산을 거친 데이터/캡처는 절대 사용하지 않는다.

| 종류 | 사용 |
|------|------|
| 회사 시스템 캡처 (CloudWatch, 모니터링 등) | ❌ |
| 회사 트래픽/메트릭 그래프 | ❌ |
| 회사 시스템에서 만든 부하 테스트 결과 캡처 | ❌ |
| 회사명/서비스명/모듈명 명시 | ❌ |
| 일반화 표현 ("팀에서 담당하는 서비스") | ✅ |
| 직접 그린 시뮬레이션 그래프 (캡션에 "(시뮬레이션)" 명시) | ✅ |
| 본인이 작성한 코드 | ✅ |

원칙: 의심스러우면 빼고, 정성적 표현으로 대체.

## AI 협업 원칙 (중요)

본문 산문은 사용자가 직접 작성한다. AI는 절대 직접 수정하지 않고, **리뷰/멘토 모드**로만 동작한다.

| 영역 | AI 수정 |
|------|---------|
| `content/posts/**/*.md`의 본문 산문 (단락, 문장, 인용문, 헤딩 텍스트) | ❌ |
| 레퍼런스 / footnote 정의 (글 끝의 `[^key]: ...` 모음) — AI 효율이 높은 영역 | ✅ |
| 본문 안의 footnote 키 삽입 (`[^key]`) | ✅ (사용자 요청 시) |
| Mermaid 다이어그램 | ✅ (변경 이유 설명 후) |
| Frontmatter (`lastmod`, `draft`, `tags` 등) | ✅ |
| 이미지 alt 텍스트 (빠진 경우 추가) | ✅ |

AI가 제공하는 것 = **재료**:
- 검증 결과 (어떤 주장이 1차 출처로 뒷받침되는지, 어떤 게 안 되는지)
- 직접 인용 가능한 문장 + 출처 URL
- 단락 구조 옵션 제안 (A/B/C 형태)
- 톤/문체 관찰
- footnote 키 매핑 같은 기계적 작업 (사용자 명시 요청 시)

**금지**: "이렇게 다시 써봤습니다" 식으로 본문 단락을 단위로 다시 쓰기. 옵션 안의 한두 줄 예시는 가능, 단락 전체 재작성은 금지.

원칙: 의심스러우면 사용자에게 묻는다. 기본값은 사용자가 직접.

## 용어 표기

| 용어 | 표기 |
|------|------|
| Auto Scaling 기능 | `Auto Scaling` |
| 수평 확장/축소 | `scale out` / `scale in` |
| 수직 확장 | `scale up` 또는 `스펙업` |
| AWS CloudWatch | `CloudWatch` |
| 알람 명칭 | `AlarmHigh`, `AlarmLow` |

## 레퍼런스 표기

```markdown
1. [제목](URL) | 저자 (년도, 출판사) — 한 줄 요약
```

본문 인용은 인라인 링크. 가능하면 anchor(`#section-id`)까지.

## 시각화 도구

| 종류 | 도구 | 비고 |
|------|------|------|
| 흐름도 / 아키텍처 | Mermaid (`flowchart`) | `graph` 대신 `flowchart` 사용 |
| 데이터 그래프 | matplotlib (Python) | 한글 폰트 `AppleGothic`, DPI 150 |
| 자유 스케치 | Excalidraw | PNG export |

- 시뮬레이션 그래프는 캡션에 **"(시뮬레이션)"** 명시 필수
- 시뮬레이션 스크립트는 page bundle 폴더에 `.py`로 같이 보관
- Mermaid subgraph `direction TB`는 일부 렌더러에서 안 먹힘 → LR로 우회

## 글 작성 흐름

```bash
# 1. 새 글 시작 (page bundle 구조)
hugo new posts/<글-슬러그>/index.md

# 2. 로컬 미리보기 (draft 포함)
hugo server -D

# 3. 발행
git add content/posts/<글-슬러그>/
git commit -m "..."
git push  # → Cloudflare Pages 자동 배포
```

### Frontmatter

```yaml
---
title: "제목"
date: 2026-04-09
lastmod: 2026-04-09
draft: true
tags: ["aws", "ecs"]
---
```

- `draft: true`로 시작 (`hugo.yaml`의 `buildDrafts: false`로 배포 안 됨)
- 발행 준비되면 `draft: false`

### 이미지 삽입

page bundle 구조이므로 같은 폴더에 두고 상대 경로로 참조:

```markdown
![alt 텍스트](image-name.png)
```

## 자주 잊는 것

- `draft: true` 확인 — 배포 전까지 false로 바꾸지 말 것
- 이미지에 alt 텍스트 명시
- 시뮬레이션 그래프 캡션에 "(시뮬레이션)"
- 본문 인용한 외부 자료를 끝의 레퍼런스 섹션에 정리
